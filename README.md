## Консольный сортировщик загрузок (Downloads Organizer)

### 1. Цель
Разработать консольное приложение, которое автоматически раскладывает файлы из папки «Загрузки» (или любой другой указанной папки) по категориям в зависимости от расширения. Утилита должна быть безопасной, корректно обрабатывать конфликты имён и предоставлять понятный отчёт о проделанной работе.

---

### 2. Обязательные функциональные требования

#### 2.1. Определение целевой папки
- При запуске программа автоматически определяет путь к папке «Загрузки» текущего пользователя:
  - Windows: `C:\Users\<Имя пользователя>\Downloads`
  - Linux/Mac: `~/Downloads`
- Если папка не найдена – программа запрашивает путь вручную (интерактивный ввод).

#### 2.2. Категории файлов (фиксированные)
Программа должна группировать файлы по следующим категориям (расширения приведены для примера, можно использовать свой набор, но он должен быть документирован):

| Категория       | Расширения                                                                 |
|-----------------|----------------------------------------------------------------------------|
| Изображения | .jpg, .jpeg, .png, .gif, .bmp, .tiff, .svg, .webp                       |
| Документы   | .pdf, .doc, .docx, .xls, .xlsx, .ppt, .pptx, .txt, .rtf, .odt, .md      |
| Архивы      | .zip, .rar, .7z, .tar, .gz, .bz2                                        |
| Аудио       | .mp3, .wav, .flac, .aac, .ogg                                           |
| Видео       | .mp4, .avi, .mkv, .mov, .wmv, .flv                                      |
| Программы   | .exe, .msi, .dmg, .deb, .rpm, .appimage                                 |
| Прочее      | Все остальные расширения (включая файлы без расширения)                   |

#### 2.3. Действия программы
1. Сканирует содержимое целевой папки (только файлы, папки игнорируются).
2. Для каждого файла определяет категорию по расширению (регистр не важен).
3. Создаёт внутри целевой папки подпапки с названиями категорий (если их ещё нет).
4. Перемещает каждый файл в соответствующую подпапку.

#### 2.4. Обработка конфликтов
- Если в папке категории уже существует файл с точно таким же именем, новому файлу добавляется суффикс `_1`, `_2` и т.д. (например, отчёт_1.pdf).
- Конфликты разрешаются автоматически, без участия пользователя.

#### 2.5. Режим предварительного просмотра (--dry-run)
- При запуске с флагом `--dry-run` или `-n` программа **не выполняет** реальное перемещение, а только выводит список файлов с указанием, в какую категорию и с каким именем они будут перемещены.
- Это позволяет пользователю проверить результат перед реальными действиями.

#### 2.6. Отчёт по завершении
После выполнения программа выводит статистику:
- Общее количество обработанных файлов.
- Количество файлов, перемещённых в каждую категорию.
- Количество конфликтов переименования (если были).

---

### 3. Технические требования
- Язык программирования – любой (Python, C#, Java, C++, Go и т.д.). Рекомендуется Python.
- Использование только стандартных библиотек – без внешних зависимостей (кроме встроенных модулей).
- Код должен быть разбит на функции (минимум 3–4 логические функции: определение категории, перемещение файла с разрешением конфликта, основной процесс, вывод отчёта).
- Обработка ошибок: программа должна корректно реагировать на:
  - Отсутствие прав доступа к файлу (пропускать файл и выводить предупреждение).
  - Невозможность создать папку (ошибка прав).
  - Некорректный путь (если задан вручную).
- Программа не должна завершаться аварийно (крашиться) при любых исключениях, связанных с файловой системой.

### 4. Критерии оценки
Оценка будет производиться по пяти основным критериям (максимум 10 баллов):

| Критерий | Описание | Макс. балл |
|----------|----------|------------|
| 1. Функциональность | Все обязательные требования реализованы корректно: определение загрузок, создание папок, перемещение, разрешение конфликтов. | 3 |
| 2. Безопасность | Наличие и корректная работа режима --dry-run. Программа не удаляет файлы, только перемещает. | 2 |
| 3. Обработка ошибок и исключений | Программа не падает при проблемах с доступом, некорректных путях, занятых файлах. Выводятся понятные сообщения. | 2 |
| 4. Чистота кода и структура | Код читаемый, осмысленные имена переменных/функций, разбивка на логические блоки. Наличие комментариев к сложным участкам. | 2 |
| 5. Отчёт и юзабилити | Вывод статистики понятен и информативен. Пользователь видит, что произошло. | 1 |
| Бонус | Реализованы дополнительные возможности (конфиг, аргумент пути, прогресс-бар и т.п.) – до +2 баллов | до +2 |

Итого: максимум 10 баллов.

### 6. Время выполнения
Спринт рассчитан на 2 часа чистого времени. Рекомендуется:
- 30 минут – проектирование и написание базового каркаса.
- 1 час – реализация основной логики (определение категорий, перемещение, конфликты).
- 30 минут – добавление `--dry-run`, отчёта, обработка ошибок и финальное тестирование.
